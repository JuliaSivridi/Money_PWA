1. Overview

Money PWA is an offline-first personal finance tracker. The user's own Google Sheets spreadsheet acts as the remote database — no dedicated backend is required. The app reads and writes directly to the Sheets API using OAuth2 access tokens obtained through Google Identity Services.

Key design decisions

  • Google Sheets as database. Removes the need for a backend server, hosting costs, and auth infrastructure beyond Google's OAuth2 flow. Trade-off: no real-time multi-device sync; each device reconciles on startup and on reconnect.
  • Offline-first via Dexie (IndexedDB). All reads come from local IndexedDB; writes go to a local queue and are flushed to Sheets asynchronously. The UI never waits for the network.
  • No persistent server-side access token. Google Identity Services issues short-lived OAuth2 access tokens (typically 3600 s). The app silently re-requests a token on startup; if the user's Google session has lapsed they see the login screen.
  • No React Router. Navigation is a single Zustand state field (selectedView). No URL-based routing exists.
  • Multi-category transactions. Each transaction carries up to two category IDs. Index 0 = primary category used for analytics; index 1 = tag only. Avoids double-counting while enabling richer labelling.
  • Sibling design system. Shares CSS custom properties and Tailwind tokens with Tasks PWA for visual identity consistency.

3. Architecture

Pattern: Zustand store layer over Dexie (IndexedDB) as primary data source; Google Sheets API as remote backend. No dedicated BFF or backend server.

Data-flow diagram

User action
    │
    ▼
React Component
    │  calls store action
    ▼
Zustand Store  ──────────────────────────────────────────►  IndexedDB (Dexie)
    │                                                             │
    │  enqueue('create'|'update'|'delete', payload)             │  bulkPut / delete
    ▼                                                             │
offlineQueue (Dexie queue table)                                 │
    │                                                             │
    │  scheduleFlush() — 800 ms debounce                        │
    ▼                                                             │
syncService.flush()                                              │
    │  dedup: latest per (entity:id:operation)                  │
    │  for each item → sheetsClient → Sheets API                │
    │  on success: markDone   on failure: markFailed, retry++   │
    ▼                                                             │
syncService.pull()                                               │
    │  fetch all 3 tables in parallel                            │
    │  upsertMany → delete orphans, bulkPut newer records       │
    └────────────────────────────────────────────────────────────►
                                                         React re-render

Write path

  1. Component calls e.g. useTransactionsStore.addTransaction(input).
  2. Store computes amount_base via convertToBase.
  3. db.transactions.add(t) — written to IndexedDB immediately.
  4. enqueue('transaction', 'create', id, t) — added to queue table.
  5. State updated optimistically: set((s) => ({ transactions: [t, ...s.transactions] })).
  6. adjustBalance called on account (also IndexedDB + queue).
  7. scheduleFlush() — 800 ms debounce → flush() → Sheets API.

Read path (on app startup)

  1. AppShell.useEffect → initialLoad().
  2. Ensure headers exist in all three sheets.
  3. flush() — push any offline queue first.
  4. pull() — parallel GET for all three ranges; each store's upsertMany.
  5. upsertMany: deletes local records absent from Sheets; bulkPuts newer records.
  6. On network failure: loadFromDb() reads from IndexedDB, UI continues offline.

Error handling

  • HTTP 401: sheetsClient catches it, calls refreshToken(), retries once.
  • Refresh fails: throws 'Session expired. Please sign in again.'
  • Any sync error: stored in syncStore.syncError; Header shows AlertCircle icon that triggers fullSync() on tap.
  • Persistent queue failures: retryCount incremented up to 5; items with retryCount >= 5 excluded from getPending().

7. Authentication & First-Launch Setup

OAuth2 scopes

email
profile
https://www.googleapis.com/auth/spreadsheets
https://www.googleapis.com/auth/drive.readonly

Sign-in flow

  1. App.tsx mounts → initAuth() (idempotent, runs once per session).
  2. initAuth injects https://accounts.google.com/gsi/client script (id gis-script).
  3. google.accounts.oauth2.initTokenClient initialised with CLIENT_ID, scopes, and login_hint = persisted user email.
  4. Back in App.tsx:
    • If isAuthenticated already true → skip.
    • Else if accessToken valid (expiry > now + 60 s) → setToken silently.
    • Else if user exists → refreshToken() silently (prompt: '', uses browser Google session).
    • If refreshToken() throws → show LoginPage.
  5. User taps "Sign in with Google" → GIS popup.
  6. GIS callback → resolveTokenRequest(token, expiresIn) → setToken → fetch /oauth2/v3/userinfo → setUser.
  7. React re-renders; AppShell mounts.

AppShell first-launch setup

  1. fetchExchangeRates(baseCurrency) — GET frankfurter.app.
  2. ensureSpreadsheet() — search Drive for name='db_money'; create with 4 sheets if not found.
  3. If new spreadsheet → seedOnboarding(): 2 accounts, 3 categories, 1 expense.
  4. initialLoad() — ensure headers, flush queue, pull all data.
  5. prefs.load() — reads settings!A1, sets baseCurrency.

Token refresh

  • Proactive: sheetsClient.getToken() refreshes if tokenExpiry < Date.now() + 60_000.
  • Reactive: HTTP 401 → refreshToken() → retry request once.

8. Synchronization / API Layer

Row-index cache

rowCache: Map<string, number> in sheetsClient.ts. Populated lazily by findRowIndex(sheet, entityId) — fetches column A, builds cache for all IDs. Cleared after flush() via invalidateRowCache(). Prevents full column scan on every update.

flush() — push local queue to Sheets

1. getPending() → items where status in ['pending','failed'] AND retryCount < 5
2. Dedup: keep only latest per key "${entityType}:${entityId}:${operationType}"
3. Mark superseded duplicates as done (deleted from queue, no API call)
4. For each latest item:
   a. markProcessing(localId)
   b. processQueueItem (append or PUT)
   c. markDone on success | markFailed(retryCount+1) on error
5. invalidateRowCache()
6. setPendingCount(await getQueueLength())

pull() — Sheets → IndexedDB

1. Parallel: fetchAllTransactions, fetchAllAccounts, fetchAllCategories
2. Parallel: upsertMany for each store
3. setLastSyncAt(now())

upsertMany pattern

existing = db.TABLE.toArray()
incomingIds = new Set(incoming.map(t => t.id))

// True sync: delete local records absent from Sheets
toDelete = existing.filter(t => !incomingIds.has(t.id))
if toDelete.length > 0: db.TABLE.bulkDelete(toDelete.map(t => t.id))

// Update only newer records
toStore = incoming.filter(item => !local || item.updated_at > local.updated_at)
if toStore.length > 0: db.TABLE.bulkPut(toStore)

set({ TABLE: (await db.TABLE.toArray()).sort(...) })

16. Key Algorithms

16.1 Transaction balance delta

computeBalanceDelta(t):
  expense       → { accountId: t.account_id,  delta: -t.amount }
  income        → { accountId: t.account_id,  delta: +t.amount }
  transfer      → { accountId: t.account_id,  delta: -t.amount,
                    toAccountId: t.to_account_id, toDelta: +t.to_amount }
  debt_lent     → t.debt_ref_id != '' ? delta: +t.amount   // repayment received
                                       : delta: -t.amount  // lent out
  debt_borrowed → t.debt_ref_id != '' ? delta: -t.amount   // repayment made
                                       : delta: +t.amount  // received

On update: reverse old delta, then apply new delta.
On delete: reverse old delta only.

16.2 Queue deduplication

Input: items[] sorted by createdAt ASC

latestMap = Map<key, item>
  key = "${entityType}:${entityId}:${operationType}"

for each item:
  if !latestMap.has(key) OR item.createdAt > latestMap.get(key).createdAt:
    latestMap.set(key, item)

latestIds = Set(latestMap.values().map(i => i.localId))

// Superseded items: delete from queue without API call
for each item NOT in latestIds: markDone(item.localId)

// Process remaining
for each item in latestMap.values():
  markProcessing → processQueueItem → markDone | markFailed(retryCount+1)

16.3 Date-group label

formatGroupLabel(date: string):
  d = parseISO(date)
  if getYear(d) === getYear(new Date()):
    datePart = format(d, 'dd MMM')         // "14 Jun"
  else:
    datePart = format(d, 'dd MMM yyyy')    // "14 Jun 2023"
  return datePart + ' · ' + format(d, 'EEEE')  // "14 Jun · Saturday"

16.4 generateId

generateId(prefix):
  return prefix + '_' + crypto.randomUUID().replace(/-/g, '').slice(0, 8)
  // e.g. generateId('txn') → 'txn_a1b2c3d4'

16.5 convertToBase

convertToBase(amount, currency, baseCurrency, rates):
  if currency === baseCurrency: return amount
  rate = rates[currency]         // 1 baseCurrency = rate foreignCurrency
  if !rate: return amount        // fallback: 1:1
  return amount / rate           // converts foreign → base

16.6 Multi-category toggle

toggle(id):
  cur = current category_ids[]
  idx = cur.indexOf(id)
  if idx === -1:
    if cur.length < 2: onChange([...cur, id])   // append (max 2)
    else:              onChange([cur[0], id])   // replace second
  else:
    onChange(cur.filter(c => c !== id))         // deselect

// cur[0] = primary (analytics, donut chart)
// cur[1] = tag only

16.7 Category delete with transfer

deleteCategory(id, transferToId):
  txns = db.transactions.where('category_ids').equals(id).toArray()
  for each txn:
    newIds = txn.category_ids
      .map(cId => cId === id ? transferToId : cId)
      .filter((cId, i, arr) => Boolean(cId) && arr.indexOf(cId) === i)  // dedup
    update txn in db + enqueue update
  db.categories.delete(id)
  setState(categories.filter(c => c.id !== id))

// Dedup prevents a transaction having the same category ID twice
// if both the deleted and transfer-target categories were already present.
